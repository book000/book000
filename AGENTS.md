# AGENTS.md - AI エージェント共通作業方針

## 目的

このファイルは、汎用 AI エージェントがこのリポジトリで作業する際の共通作業方針を定義します。

## 基本方針

### 言語ルール

- **会話言語**: 日本語
- **コード内コメント**: 日本語
- **エラーメッセージ**: 英語
- **日本語と英数字の間**: 半角スペースを挿入

### コミット規約

- **コミットメッセージ**: [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) に従う
  - 形式: `<type>(<scope>): <description>`
  - `<description>` は日本語で記載
  - 例: `feat: メトリクス生成ワークフローを追加`

## 判断記録のルール

重要な判断を行う際は、以下を記録してください：

1. **判断内容**: 何を決定したか
2. **代替案**: どのような選択肢があったか
3. **採用理由**: なぜその選択肢を選んだか
4. **前提条件**: 判断の前提となる条件
5. **不確実性**: 確実でない事項や仮定

**重要**: 前提・仮定・不確実性を明示し、仮定を事実のように扱わないこと。

## 開発手順（概要）

このリポジトリで作業する際の基本的な手順：

1. **プロジェクト理解**
   - README.md を読み、プロジェクトの目的と構造を理解する
   - ワークフローファイルを確認し、自動化の仕組みを把握する

2. ~~**依存関係インストール**~~
   - このリポジトリはプロフィールリポジトリのため、依存関係のインストールは不要

3. **変更実装**
   - README.md の更新、ワークフローの追加・修正など
   - Conventional Commits に従ってコミット

4. **テストと確認**
   - ワークフローを変更した場合は、手動実行で動作を確認
   - bash スクリプトを変更した場合は、ローカル環境でテスト

## セキュリティ / 機密情報

- **認証情報のコミット禁止**: API キーやトークンをコードにハードコードしない
- **ログへの機密情報出力禁止**: ログに個人情報や認証情報を出力しない
- **GitHub Secrets の使用**: トークンは GitHub Secrets に保存し、ワークフローで参照する

## リポジトリ固有

### プロジェクト概要

- **目的**: book000 (Tomachi) の GitHub プロフィールリポジトリを管理
- **主な機能**:
  - GitHub メトリクスの自動生成（hourly）
  - 月次更新 Issue の自動作成（monthly）
  - プロフィール README の管理

### 技術スタック

- **言語**: Markdown, Bash, YAML
- **GitHub Actions**: ワークフローの自動実行
- **使用ツール**: `gh` CLI、`jq`、`sed`

### 重要な注意事項

- このリポジトリは GitHub プロフィールページ（`https://github.com/book000`）として表示される。
- README.md の変更は即座にプロフィールページに反映される。
- ワークフローは2つ：
  - `metrics.yml`: GitHub メトリクスの SVG を hourly で生成
  - `monthly-update.yml`: 月初に Renovate PR の一覧を含む Issue を作成
- MIT ライセンスで公開されており、他者がプロフィール README の参考として利用可能。
